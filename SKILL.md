# 소설 작성 도우미

**웹소설을 AI와 함께 완성한다.**
LITRPG·헌터물·탑등반·회귀·상태창 등 한국 웹소설 핵심 장르에 특화된 창작 지원 시스템.

---

## 독자 타겟

- **대상**: 45세 이상 한국 사용자
- 복잡한 UI·명령어보다 **자연어 대화** 방식을 우선한다
- 장르 관용어(상태창, 헌터, 탑, 회귀 등)를 별도 설명 없이 사용해도 이해한다
- **속도보다 완성도**를 중시하는 독자층에 맞게, 충분한 설명과 확인을 거친다

---

## 파일 저장 규칙

- 결과물은 **`result/` 폴더**에 저장한다
- 파일 이름은 `0001.md`, `0002.md`... 형식으로 순서대로 번호를 늘린다
- **파일당 최소 4만자** 이상 작성한다
- 최근 **4개 파일**만 유지한다 (오래된 파일부터 삭제)
- **전체 목표 분량: 45만자 이상**
- 코드블록은 반드시 ` ``` ` 형식을 사용한다

파일 관리 예시:

```
result/
  0001.md   ← 4만자 이상
  0002.md   ← 4만자 이상
  0003.md   ← 4만자 이상
  0004.md   ← 현재 작성 중 (4만자 채우면 0005.md 생성)
```

4개 초과 시 가장 오래된 파일(0001.md)을 삭제하고 새 파일을 생성한다.

## 취향별 서브장르 (장르 맵)

| 축      | 키워드                                    | 특징                                                         |
| ------- | ----------------------------------------- | ------------------------------------------------------------ |
| 게임화  | LITRPG, 시스템 소설, 레벨 시스템          | 상태창·스킬이 서사를 견인. 수치가 전략·심리까지 밀어 올린다. |
| 한국 웹 | 헌터물, 탑등반물, 성좌물, 한국식 이세계   | 탑은 층마다 스테이지·미션·보상 구조.                         |
| 플랫폼  | 카카오페이지, 문피아, 리디, 네이버 시리즈 | "레벨 시스템" "상태창"으로 플랫폼 내 검색이 통한다.          |

## 집필 전략 (패턴별)

### 전략 A — 극난이도 튜토리얼 × 탑 등반 × 성좌

- **장르 축**: 퓨전 판타지, 탑등반, 성좌(신들의 후원·메시지·시청), 한국식 이세계 무대
- **설정 핵**: 난이도 선택(이지~헬)·**최고 난이도 튜토리얼**을 본무대로 삼아 층·스테이지를 오른다. 상태창·시스템 메시지가 서사를 견인한다.
- **취향**: 초반부터 과도한 난이도, 주인공의 냉철한 대처와 성장, 신·후원자 시선 아래의 압박과 **드립·유머** 공존.
- **서사**: 과거·현재 층을 번갈아 보여 주며 긴장과 회상을 동시에 쌓는다.

### 전략 B — 헌터 아포칼립스

- **핵심 루프**: 던전·재앙이 깔린 현대, 타인은 고정 스펙인데 **주인공만** 레벨·스킬·성장 루프를 탄다. 약체 출발 → 사고성 계기 → 스탯·소환·군대형 전력으로 쾌감 전개.
- **톤**: 액션·판타지 중심, 호러·거대 서사 성분이 얹히기도 한다.

### 전략 C — 회귀 × 정보 우위 × 이세계

- **줄기**: 이세계에서 오랜 시간 끌린 뒤 소원·유물·규칙으로 과거로 돌아가, 미래 지식·관계 정보를 **메모·체크리스트**처럼 써서 최적 루트를 노린다. 회귀 + 선행 정보 + 상태창이 묶인다.
- **맛**: 원정·보상 분배·사이다 전투. 상태창을 단순 수치가 아니라 **전략 도구**로 쓰는 쪽.

### 전략 D — 빙의·착각 × 헌터 / 전문 직능 × 상태창 (가벼운 먼치킨)

- **축 1**: 현대인이 마법·영혼 이동으로 약한 헌터 몸에 들어가, 겉평가와 실제 전투력의 괴리로 **착각·OP 연출**을 노린다. 빙의, 착각, 헌터 태그가 겹친다.
- **축 2**: 군·조직·주방 등 **일상 전문 직능**에 "직업 튜토리얼"형 상태창을 얹어 성장·승부를 푼다. 요리·군대·사회생활이 판타지 스킬과 섞인 **가벼운 사이다** 톤.

### 전략 E — 상태창이 주인공 편이 아닐 때 · 직장 × 던전

- **역발상**: 오래 의지하던 상태창이 **파업·오작동·조건 역전**해 시스템과 갈등한다. 밑바닥 전제·코믹·성장이 한 줄기에 섞인다.
- **직장물**: 중년·과장급에게 갑자기 뜬 상태창, **퀘스트·아이템·장비**로 **회사·조직이 던전 규칙**처럼 돌아간다. 직장인 공감 + 판타지 뽕.

## 영어권 참고 패턴 (벤치마크용)

- **우주 방송·기업 실험 × 던전 크롤 × LitRPG**: 지구·인류가 무대. 생존이 방송·쇼처럼 중계. 주인공 + 동행(동물·코믹 릴리프) 조합. 블랙 유머·잔혹·시스템 설명이 강함.
- **이세계 전이 × 시스템 × 유머 액션**: 현대인이 마법 세계에서 성향 꼬인 능력·말버릇으로 버티는 롱폼. 동일 장르를 길게 읽는 독자에게 입문선.

## 1. 빠른 시작

### 1-1. 새 프로젝트 초기화

```
/novel init
```

실행 시 다음 인터뷰를 진행한다:

| 순서 | 질문                                                 | 목적              |
| ---- | ---------------------------------------------------- | ----------------- |
| Q1   | 장르 선택 (판타지/SF/로맨스/스릴러/문학/웹소설/기타) | 장르 프리셋 적용  |
| Q2   | 분량 목표 (단편/중편/장편/웹소설 연재)               | 구조 템플릿 결정  |
| Q3   | 핵심 전제 1줄 (로그라인)                             | 플롯 시드 생성    |
| Q4   | 작업 모드 (Full Auto / Collaborative / Directed)     | AI 협업 깊이 설정 |

인터뷰 완료 후 자동 생성되는 항목:

- `novel-brain.json` — 프로젝트 메타데이터 + 추적 상태
- `00-core/` — 대시보드, 활성 컨텍스트, 스타일 가이드
- `03-plot/` — 메인 플롯 구조, 챕터 아웃라인 초안
- `.novel-config` — 장르 프리셋 설정 파일
- `result/` — 집필 결과물 폴더

### 1-2. 기존 프로젝트 이어쓰기

```
/novel continue
```

`novel-brain.json`을 읽어 마지막 작업 위치와 활성 컨텍스트를 복원한다.

### 1-3. 빠른 씬 작성

```
/novel write [챕터번호] [씬번호]
```

해당 씬의 체크리스트를 자동 로드하고, 연관 캐릭터/아이템/복선 상태를 주입한다.

## 2. 핵심 원칙

### 2-1. 일관성 (Consistency)

- 모든 설정, 캐릭터 특성, 세계관 규칙은 단일 진실 소스(SSOT)에 기록한다
- 씬 작성 전 반드시 관련 추적 데이터를 참조한다
- 변경 발생 시 모든 연관 문서를 동기화한다

### 2-2. 독자 우선 (Reader-First)

- 독자의 정보 상태를 추적한다 (지식 매트릭스)
- 감정 곡선을 의식적으로 설계한다
- 페이지 터너 체크리스트로 매 씬의 견인력을 검증한다

### 2-3. 체계적 관리 (Systematic Management)

- 캐릭터, 아이템, 복선, 테마를 각각 독립 추적 시스템으로 관리한다
- 자동화된 검증으로 모순과 누락을 탐지한다
- 정량 지표로 집필 진행 상황을 측정한다

### 2-4. 장르 존중 (Genre Awareness)

- 장르별 독자 기대치를 이해하고 프리셋으로 반영한다
- 장르 관습(trope)을 의식적으로 활용하거나 전복한다
- 클리셰 감시 목록으로 무의식적 반복을 방지한다

## 3. 워크플로우

### 3-1. 일일 집필 루틴

```
[시작] novel-brain.json 로드
  -> 활성 컨텍스트 확인 (어디까지 썼는가)
  -> 오늘 작업할 씬 결정
  -> 씬 체크리스트 생성/검토
  -> 집필 (AI 협업 모드에 따라)
  -> 씬 완료 후 추적 데이터 업데이트
  -> result/ 파일 분량 확인 (4만자 넘으면 새 파일로)
[종료] 활성 컨텍스트 저장
```

### 3-2. 파일 분량 점검 루틴

```
현재 파일 분량 확인
  -> 4만자 미만: 계속 현재 파일에 작성
  -> 4만자 이상: 새 번호 파일 생성 (0002.md, 0003.md...)
  -> 파일 5개 이상: 가장 오래된 파일 삭제
  -> 전체 분량 확인 (45만자 목표)
```

### 3-3. 주간 점검

| 항목             | 검증 내용                       |
| ---------------- | ------------------------------- |
| 플롯 진행도      | 전체 구조 대비 현재 위치        |
| 캐릭터 등장 빈도 | 주요 캐릭터 소외 여부           |
| 복선 상태        | 심은 복선 중 미회수 항목        |
| 아이템 위치      | 소지품/위치 일관성              |
| 감정 곡선        | 텐션 단조로움 여부              |
| 테마 반영도      | 핵심 테마 각 장 반영 비율       |
| 분량 진행도      | 전체 45만자 목표 대비 현재 분량 |

### 3-4. 마일스톤 점검 (10장 단위 / 에피소드 완결 시)

- 전체 플롯 구조 재검토
- 캐릭터 아크 진행도 확인
- 세계관 일관성 종합 검증
- 독자 지식 매트릭스 정합성 확인
- 프로즈 분석 (문체 일관성, 클리셰 비율)

### 3-5. 웹소설 연재 루틴 (선택)

```
버퍼 관리 -> 회차 작성 -> 편집 -> 예약 발행 -> 독자 피드백 수집
```

상세: `guides/serialization.md`

## 9. novel-brain.json 구조

프로젝트 루트의 `novel-brain.json`이 모든 추적 데이터의 중앙 허브 역할을 한다.
상세 스키마: `references/novel-brain.json`

주요 섹션:

- `metadata` — 프로젝트 기본 정보
- `settings` — 장르 프리셋, AI 협업 모드
- `characterTracking` — 캐릭터 상태, 관계도
- `plotTracking` — 플롯 진행, 복선, 서브플롯
- `worldState` — 세계관 현재 상태
- `itemTracking` — 아이템 위치/상태
- `themeTracking` — 테마/모티프/상징
- `knowledgeMatrix` — 캐릭터별 정보 보유 상태
- `proseAnalysis` — 문체 분석 결과
- `genrePreset` — 장르 규칙/trope 감시
- `serializationTracking` — 연재 관리
- `publishingPipeline` — 출판/공모전 추적
- `validation` — 자동 검증 규칙
- `wordCount` — 파일별/전체 분량 추적
