# 세계관 구축 가이드

소설의 세계관을 체계적으로 설계하고, 일관성을 유지하며, 몰입감을 높인다.

---

## 1. 세계관 설계 계층

### 1-1. 거시 세계 (Macro World)

| 요소 | 정의 항목 |
|------|----------|
| 물리 법칙 | 현실 준수 / 변형 규칙 (마법, SF 기술) |
| 지리 | 대륙, 국가, 도시, 지형 |
| 역사 | 주요 사건, 시대 구분, 현재 시점 |
| 정치 | 권력 구조, 세력 관계, 갈등 |
| 경제 | 화폐, 무역, 자원, 계층 |
| 문화 | 종교, 관습, 언어, 예술 |
| 기술 | 기술 수준, 발전 방향, 제약 |

### 1-2. 미시 세계 (Micro World)

| 요소 | 정의 항목 |
|------|----------|
| 장소 | 구체적 장소의 물리적 특성 |
| 오감 묘사 | 시각/청각/후각/촉각/미각 데이터 |
| 일상 | 식사, 이동, 통신, 노동 방식 |
| 사회 규범 | 예절, 금기, 의복, 인사 |

---

## 2. 마법/기술 체계 설계

### 2-1. 샌더슨의 마법 법칙

**제1법칙**: 마법으로 갈등을 해결하는 능력은 독자가 그 마법을 이해하는 정도에 비례한다.
- 경성 마법 (Hard Magic): 명확한 규칙, 비용, 제한 -> 퍼즐형 해결 가능
- 연성 마법 (Soft Magic): 모호한 규칙 -> 경이감 연출, 문제 해결에 사용 불가

**제2법칙**: 제한이 힘보다 중요하다.
- 마법의 비용/대가를 반드시 정의한다
- 비용이 없는 마법은 긴장을 파괴한다

**제3법칙**: 새로운 마법을 추가하기 전에 기존 마법을 확장하라.
- 체계의 복잡성을 억제한다
- 기존 규칙의 새로운 활용을 탐구한다

### 2-2. 체계 정의 템플릿

```
이름: [체계 이름]
원리: [어떻게 작동하는가]
에너지원: [동력/연료]
사용 조건: [누가, 언제, 어떻게]
비용/대가: [사용 시 치르는 것]
한계: [할 수 없는 것]
사회적 위치: [체계 사용자의 사회적 위상]
역사: [발견/발전 과정]
```

## 3. 장소 관리

### 3-1. 장소 등록

모든 주요 장소는 `01-world/geography.md`에 등록한다:
- 이름, 위치, 유형
- 연결된 장소 (이동 가능 경로)
- 이동 소요 시간 (이동 일관성 검증용)

### 3-2. 오감 묘사 데이터

`01-world/location-sensory.md`에 장소별 감각 데이터를 기록한다:

```
[장소명]
- 시각: [빛, 색, 형태, 크기]
- 청각: [소리, 정적, 반향]
- 후각: [향기, 악취, 미묘한 냄새]
- 촉각: [온도, 질감, 습도, 바람]
- 미각: [해당 장소 고유의 맛 -- 음식, 공기, 물]
```

씬 작성 시 해당 장소의 감각 데이터를 컨텍스트로 주입한다.

## 4. 시간축 관리

### 4-1. 작품 내 달력

세계관 고유의 달력 체계를 정의한다 (현실 기반이면 실제 달력 사용):
- 연/월/일/시간 단위
- 계절 체계
- 주요 기념일/축제

### 4-2. 타임라인 검증

- 사건 간 경과 시간이 논리적인지 확인
- 이동 시간이 거리/교통수단과 일치하는지 확인
- 동시에 다른 장소에서 벌어지는 사건의 정합성 확인
- 캐릭터의 나이 변화가 일관적인지 확인

---

## 5. 세계관 일관성 검증

### 5-1. 자동 검증 항목

novel-brain.json의 validation 규칙 중 세계관 관련:
- `world-rules`: 정의된 규칙 위반 감지
- `geography`: 이동 거리/시간 불일치 감지
- `timeline`: 시간축 모순 감지

### 5-2. 수동 검증 체크리스트

- [ ] 마법/기술 체계의 규칙이 일관적으로 적용되었는가
- [ ] 사회 규범이 장면마다 일관적인가
- [ ] 기술 수준에 어울리지 않는 물건/행동이 없는가
- [ ] 계절/날씨가 타임라인과 일치하는가
- [ ] 화폐/경제 체계가 현실적인가

---

## 6. 아이스버그 원칙

전체 세계관의 10-20%만 작품에 직접 드러낸다.
나머지 80-90%는 배경으로 존재하며, 세계의 깊이감을 만든다.

규칙:
- 설정을 설명하지 말고 보여줘라 (Show, Don't Tell)
- 캐릭터의 행동/대사를 통해 세계관을 자연스럽게 드러내라
- 정보 덤프(Info Dump)를 피하라 -- 한 번에 3문장 이상의 세계관 설명 금지
- 독자가 알아야 할 때만 공개하라 (Just-in-Time Exposition)

---

## 7. novel-brain.json 연동

세계관 데이터는 `worldState` 섹션에서 관리한다.
- `rules`: 물리/마법/사회/경제 규칙
- `locations`: 장소 등록소
- `calendar`: 시간 체계
- `currentGlobalState`: 현재 세계 상태
