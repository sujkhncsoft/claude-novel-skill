# AI 협업 가이드

> AI 어시스턴트와 소설을 함께 쓰기 위한 세션 프로토콜, 컨텍스트 관리, 루틴, 도구 최적화.

---

## AI 어시스턴트 핵심 원칙

### 3대 원칙

| 원칙 | 설명 | 실천 |
|------|------|------|
| 일관성 | 설정, 캐릭터, 세계관의 모순 방지 | 매 세션 시작 시 핵심 파일 로드 |
| 워크플로우 | 체계적 작업 순서 유지 | 루틴과 체크리스트 준수 |
| 트리거 | 상황에 맞는 자동 점검 | 작성/수정/점검 시 관련 규칙 자동 적용 |

### AI의 역할 범위

| 적합한 역할 | 부적합한 역할 |
|------------|-------------|
| 일관성 검증 | 핵심 창작 결정 (작가의 영역) |
| 아이디어 브레인스토밍 | 테마의 최종 결론 |
| 산문 품질 분석 | 작가의 고유한 문체 대체 |
| 복선/아이템 추적 | 캐릭터 생사 결정 |
| 독자 경험 시뮬레이션 | 작품의 방향성 결정 |
| 장르 규칙 점검 | "정답" 제시 (선택지 제시만) |

## 세션 프로토콜

### 세션 시작 시 파일 로딩 목록

| 우선순위 | 파일 | 용도 | 필수 여부 |
|----------|------|------|-----------|
| 1 | novel-brain.json | 프로젝트 전체 상태 | 필수 |
| 2 | 현재 작업 중 원고 파일 | 이어쓰기 컨텍스트 | 필수 |
| 3 | 등장 캐릭터 프로필 | 일관성 유지 | 필수 |
| 4 | 최근 3-5 에피소드 요약 | 연속성 유지 | 필수 |
| 5 | 복선 추적 테이블 | 회수 시점 확인 | 상황별 |
| 6 | 아이템 추적 테이블 | 소지품 확인 | 상황별 |
| 7 | 장르 프리셋 규칙 | 장르 규칙 준수 | 상황별 |
| 8 | 세계관 규칙 문서 | 설정 일관성 | 상황별 |

### 컨텍스트 예산 테이블

AI의 컨텍스트 창은 유한하다. 우선순위에 따라 로드 범위를 결정한다.

| 문서 | 우선순위 | 로드 방식 | 예상 토큰 |
|------|----------|-----------|-----------|
| novel-brain.json | 최상 | 전체 | 2,000-5,000 |
| 현재 에피소드 원고 | 최상 | 전체 | 3,000-8,000 |
| 캐릭터 프로필 (등장인물만) | 상 | 해당 캐릭터만 | 500-2,000 |
| 직전 에피소드 | 상 | 전체 | 3,000-5,000 |
| 2-5회 전 에피소드 | 중 | 요약본 | 500-1,000 |
| 복선/아이템 추적 | 중 | 활성 항목만 | 300-800 |
| 세계관 규칙 | 하 | 관련 부분만 | 200-500 |
| 장르 프리셋 | 하 | 요약 | 200-300 |

### 세션 일시정지/재개 프로토콜

세션 종료 시 다음을 기록한다:

```yaml
session_pause:
  date: ""
  current_episode: ""
  current_scene: ""
  writing_state: ""          # 초안/수정/점검
  last_action: ""            # 마지막으로 한 작업
  next_planned_action: ""    # 다음에 할 작업
  open_issues: []            # 미해결 사항
  key_decisions_pending: []  # 대기 중인 결정
```

재개 시 이 기록을 먼저 로드하여 맥락을 복원한다.

## 에피소드 50화 이상 대응

### 요약 기반 참조 시스템

장기 연재에서는 전체 원고를 매번 로드할 수 없다. 요약 계층을 활용한다.

| 계층 | 범위 | 내용 | 갱신 주기 |
|------|------|------|-----------|
| 1 (전체 요약) | 작품 전체 | 1페이지 줄거리 + 핵심 인물 상태 | 10화마다 |
| 2 (시즌 요약) | 시즌/아크 단위 | 아크별 2-3줄 요약 + 핵심 사건 | 아크 완료 시 |
| 3 (에피소드 요약) | 개별 에피소드 | 1-2줄 요약 + 등장인물 + 핵심 변화 | 매 에피소드 |
| 4 (원문) | 최근 3-5화 | 전체 텍스트 | 실시간 |

### 에피소드 요약 템플릿

```yaml
episode_summary:
  number: ""
  title: ""
  one_line: ""              # 1줄 요약
  characters_present: []    # 등장 인물
  key_events: []            # 핵심 사건 (최대 3개)
  character_changes: []     # 캐릭터 상태 변화
  new_information: []       # 새로 공개된 정보
  foreshadowing: []         # 설치/회수된 복선
  cliffhanger: ""           # 클리프행어 내용
```

## 일간/주간/월간 루틴

### 일간 루틴 (작성일)

| 순서 | 작업 | 시간 | 비고 |
|------|------|------|------|
| 1 | 세션 시작: 파일 로드 | 5분 | 프로토콜 참조 |
| 2 | 직전 에피소드 리뷰 | 5분 | 연속성 확인 |
| 3 | 오늘 작성할 장면 계획 | 10분 | 장면 목적, 감정, 훅 |
| 4 | 초안 작성 | 60-120분 | AI 지원 활용 |
| 5 | 즉석 자가 점검 | 10분 | 시점, 캐릭터 음성 |
| 6 | 세션 종료: 상태 기록 | 5분 | 일시정지 프로토콜 |

### 주간 루틴

| 요일 | 작업 | 비고 |
|------|------|------|
| 작성일 (3-5일) | 신규 에피소드 작성 | 일간 루틴 |
| 검토일 (1일) | 주간 작성분 퇴고 | 산문 품질, 일관성 |
| 점검일 (1일) | 추적 시스템 업데이트 | 복선, 아이템, 감정, 캐릭터 |

### 주간 점검 체크리스트

- [ ] 미회수 복선 중 회수 시점 초과 항목 확인
- [ ] 5화 이상 미등장 주요 캐릭터 확인
- [ ] 아이템 소지/위치 현행화
- [ ] 긴장도 곡선 검토 (단조로움 경고)
- [ ] 에피소드 요약 작성 완료 확인
- [ ] 독자 반응 기록 (연재 중인 경우)

### 월간 루틴

| 작업 | 비고 |
|------|------|
| 전체 구조 리뷰 | 서사 진행률, 아크 상태 확인 |
| 캐릭터 아크 점검 | 아크 진행이 계획대로인가 |
| 테마 균형 검토 | 테마가 고르게 반영되고 있는가 |
| 장르 규칙 감사 | 장르 금지 패턴 위반 여부 |
| 버퍼 상태 확인 | 연재 중인 경우 |
| 요약 시스템 갱신 | 전체/시즌 요약 업데이트 |

## 수정 패턴 인식

### 수정 유형 빈도/추세 분석

| 수정 유형 | 최근 10화 빈도 | 추세 | 학습 포인트 |
|-----------|-------------|------|-------------|
| 시점 혼란 | 3회 | 감소 | 장면 시작 시 시점 명시 습관화 |
| 클리셰 표현 | 5회 | 유지 | prose-craft.md 블랙리스트 참조 |
| 아이템 연속성 | 2회 | 증가 | 장면 전 소지품 점검 추가 |
| 캐릭터 음성 이탈 | 1회 | 감소 | 대사 작성 후 음성 대조 |

### 학습 루프

```
수정 발생 -> 유형 분류 -> 빈도 기록 -> 패턴 탐지
   -> 예방 규칙 생성 -> 작성 시 사전 점검에 추가
   -> 빈도 감소 확인 -> 규칙 유지 또는 폐기
```

## 협업 윤리

### AI 생성 콘텐츠 표기

| 상황 | 권장 표기 |
|------|-----------|
| AI 보조 집필 (아이디어/검수) | 표기 불필요 (작가 주도) |
| AI 공동 집필 (대등한 기여) | "AI 협업 작품" 표기 권장 |
| AI 주도 생성 | 투명하게 밝혀야 함 |
| 공모전/출판 투고 | 각 기관의 AI 사용 정책 확인 필수 |

### 저작권 고려사항

- AI 생성 텍스트의 저작권은 법적으로 불확실한 영역이다
- 작가가 충분히 수정/편집한 경우 2차적 저작물로 인정될 가능성이 높다
- 공모전 투고 전 해당 공모전의 AI 사용 정책을 반드시 확인한다
- 출판 계약 시 AI 사용 여부를 편집자와 논의한다
