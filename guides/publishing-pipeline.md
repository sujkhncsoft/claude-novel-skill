# 출판 파이프라인 가이드

원고 완성 후 공모전 투고, 출판사 투고, 자가출판까지의 전체 과정을 관리한다.

---

## 1. 출판 경로

### 1-1. 경로별 특성

| 경로 | 진입 난이도 | 수익 | 통제권 | 소요 시간 |
|------|-----------|------|--------|----------|
| 문학상/공모전 | 높음 | 상금 + 출판 기회 | 낮음 | 6-12개월 |
| 출판사 직접 투고 | 높음 | 인세 8-12% | 중간 | 3-12개월 |
| 문학 에이전시 | 중간 | 인세 (에이전트 15%) | 중간 | 6-18개월 |
| 웹소설 플랫폼 | 낮음 | 조회/구독 수익 | 높음 | 즉시 |
| 자가출판 (전자) | 낮음 | 70% 로열티 | 최고 | 1-3개월 |
| 자가출판 (종이) | 중간 | 다양 | 최고 | 3-6개월 |

---

## 2. 원고 준비

### 2-1. 퇴고 단계

```
1차 퇴고: 구조적 수정 (플롯, 캐릭터 아크, 페이싱)
2차 퇴고: 씬 레벨 수정 (불필요 씬 삭제, 순서 조정)
3차 퇴고: 문장 레벨 수정 (문체, 표현, 클리셰)
4차 퇴고: 교정 (오타, 맞춤법, 문법)
```

### 2-2. 베타 리더

- 최소 3명의 베타 리더 확보
- 다양한 배경 (타깃 독자, 작가, 일반 독자)
- 구조화된 피드백 양식 제공 (`references/templates/project/07-business/beta-feedback.md`)
- 피드백 수렴 -> 최종 수정

---

## 3. 공모전/문학상

### 3-1. 주요 공모전

| 공모전 | 마감 | 분량 | 장르 |
|--------|------|------|------|
| 한국일보 문학상 | 매년 상반기 | 200매 내외 | 순문학 |
| 문학동네 소설상 | 매년 6월 | 장편 | 순문학 |
| 창비 장편소설상 | 매년 하반기 | 장편 | 순문학 |
| 대산 대학문학상 | 매년 | 중편 | 순문학 |
| 웹소설 공모전 | 플랫폼별 상시 | 다양 | 장르 소설 |

### 3-2. 투고 관리

`publishingPipeline.submissions`에 각 투고를 기록:
- 대상, 마감일, 제출일, 현재 상태
- 요구 형식 (매수, 폰트, 행간)
- 동시 투고 가능 여부 확인

## 4. 시놉시스/기획서 작성

### 4-1. 시놉시스 구조

```
1줄 로그라인 (50자 이내)
단편 시놉시스 (500자): 핵심 갈등 + 결말
장편 시놉시스 (2000자): 3막 구조 + 캐릭터 아크 + 결말
작가 소개 (300자): 경력, 수상 이력
```

### 4-2. 쿼리 레터

출판사/에이전시에 보내는 투고 편지:
- 훅: 첫 문장에서 관심 잡기
- 시놉시스: 핵심 갈등 + 왜 이 이야기인가
- 비교 작품: "X와 Y의 만남" 형식
- 작가 소개: 관련 경력만 간결히
- 실례: "원고를 보내도 될까요"

템플릿: `references/templates/project/07-business/synopsis-query.md`

## 5. 원고 포맷

### 5-1. 국내 출판사 표준

```
폰트: 바탕 (또는 출판사 지정 폰트)
크기: 10pt
행간: 160%
용지: A4
여백: 상하좌우 30mm
파일: HWP 또는 DOCX
```

### 5-2. 전자책 (e-book)

```
포맷: EPUB 3.0
표지: 1600x2560px (2:3 비율)
목차: 자동 생성
메타데이터: ISBN, 저자, 출판사
```

## 6. 자가출판

### 6-1. 전자책 플랫폼

| 플랫폼 | 로열티 | 독점 여부 |
|--------|--------|----------|
| 리디 셀렉트 | 정산 방식 다양 | 선택적 |
| 유페이퍼 | 70% | 비독점 |
| 브런치북 | 출판 계약 | 선택적 |
| 아마존 KDP | 35-70% | 선택적 |

### 6-2. 자가출판 체크리스트

- [ ] 최종 퇴고 완료
- [ ] 표지 디자인 (캔버스 디자인 스킬 활용 가능)
- [ ] ISBN 발급 (한국: 국립중앙도서관)
- [ ] 메타데이터 준비 (제목, 부제, 장르, 키워드)
- [ ] 가격 결정
- [ ] 판매 페이지 설명문 작성
- [ ] 출간 후 마케팅 계획

## 7. novel-brain.json 연동

출판 데이터는 `publishingPipeline` 섹션에서 관리한다.
- `status`: 현재 원고 단계
- `submissions`: 투고 이력
- `manuscriptFormats`: 포맷 설정
- `synopsis`: 시놉시스 버전 관리
- `betaReaders`: 베타 리더 피드백
- `editingRounds`: 퇴고 이력

`/novel publish` 명령으로 출판 파이프라인을 관리할 수 있다.
