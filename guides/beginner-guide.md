# 초보자 가이드

처음 소설을 쓰는 사용자를 위한 단계별 안내.
이 가이드만으로도 첫 소설 프로젝트를 시작할 수 있다.

---

## 1. 소설 쓰기 전에 알아야 할 것

### 1-1. 완벽한 첫 소설은 없다

- 첫 소설은 연습이다. 완성이 목표다.
- "좋은 글쓰기"보다 "꾸준한 글쓰기"가 먼저다.
- 수정은 초고 완성 후에 한다. 초고 중 수정은 완성을 방해한다.

### 1-2. 최소 준비물

```
1. 아이디어 하나 (한 줄 로그라인)
2. 주인공 하나 (이름 + 원하는 것 + 방해물)
3. 목표 분량 (단편: 2만자, 중편: 5만자, 장편: 20만자+)
4. 쓸 시간 (매일 30분이면 충분)
```

---

## 2. 5단계 시작법

### 단계 1: 아이디어 정리

한 줄로 이야기를 요약한다 (로그라인):

```
"[주인공]이 [사건]을 겪고 [목표]를 향해 [장애물]과 싸운다."
```

예시:
- "평범한 고등학생이 이세계로 소환되어 마왕을 물리치러 간다."
- "형사가 연쇄 실종 사건을 추적하며 자신의 과거와 마주한다."

### 단계 2: 캐릭터 만들기

최소 3명:
- 주인공: 이름, 나이, 원하는 것, 약점
- 조력자: 주인공을 돕는 사람, 관계
- 적대자: 주인공을 방해하는 존재, 이유

### 단계 3: 구조 잡기

가장 단순한 3막 구조:
```
1막 (처음 25%): 주인공 소개 + 사건 발생
2막 (중간 50%): 문제 해결 시도 + 점점 어려워짐
3막 (마지막 25%): 최종 대결 + 결말
```

### 단계 4: 첫 장 쓰기

첫 장에서 할 일:
- 주인공을 등장시킨다
- 주인공의 일상을 보여준다
- 일상을 깨뜨리는 사건을 일으킨다
- 독자가 "다음에 뭐가 일어날까" 궁금하게 끝낸다

### 단계 5: 계속 쓰기

- 매일 같은 시간에 쓴다 (습관 형성)
- 목표 분량을 정한다 (하루 500자도 좋다)
- 막히면 다음 씬으로 건너뛴다
- AI에게 도움을 요청한다 (`/novel suggest`)

---

## 3. 초보자가 자주 하는 실수

| 실수 | 해결법 |
|------|--------|
| 설정만 만들고 안 쓴다 | 설정은 필요한 만큼만. 쓰면서 보충 |
| 초고를 고치면서 쓴다 | 초고는 일단 끝까지. 수정은 나중에 |
| 너무 긴 설명/묘사 | 행동과 대화로 보여주기 |
| 모든 것을 한 번에 공개 | 정보를 조금씩 풀기 |
| 캐릭터가 다 비슷하다 | 각자 다른 말투/가치관 부여 |
| 갈등 없이 진행 | 매 씬에 뭔가 원하고 + 방해 받아야 |

---

## 4. AI와 함께 쓰기 기초

### 4-1. AI에게 요청하면 좋은 것

| 요청 | 명령어 |
|------|--------|
| "다음 전개 아이디어 줘" | `/novel suggest` |
| "이 캐릭터가 이 상황에서 뭐라고 할까" | `/novel voice [이름]` |
| "내가 쓴 것 문체 분석해줘" | `/novel prose` |
| "복선 아이디어 줘" | `/novel brainstorm 복선` |
| "이 장 검수해줘" | `/novel check` |

### 4-2. AI에게 맡기지 말아야 할 것

- 핵심 캐릭터 생사 결정
- 메인 플롯의 방향 전환
- 작품의 테마적 결론
- 이것들은 작가가 결정하는 것이다

## 6. 첫 프로젝트 시작

```
/novel init
```

위 명령을 입력하면 인터뷰가 시작된다.
장르, 분량, 로그라인, 작업 모드를 선택하면
프로젝트가 자동으로 세팅된다.

초보자 권장 설정:
- 장르: 가장 좋아하는 장르
- 분량: 중편 (5만자)
- 모드: Collaborative (AI와 함께)
