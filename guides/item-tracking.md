# 아이템 추적 가이드

소설 내 물건, 소지품, 중요 오브젝트의 위치와 상태를 체계적으로 추적한다.
"그 칼은 2장에서 버렸는데 5장에서 다시 쓰고 있다" 같은 오류를 방지한다.

---

## 1. 아이템 분류

### 1-1. 중요도별 분류

| 등급 | 설명 | 추적 수준 | 예시 |
|------|------|----------|------|
| 핵심 | 플롯에 직접 영향 | 매 씬마다 위치/상태 추적 | 반지, 편지, 열쇠, 무기 |
| 중요 | 캐릭터/분위기에 영향 | 등장 시마다 상태 확인 | 의복, 가방, 지갑 |
| 일반 | 장면 디테일용 | 모순만 방지 | 음식, 도구, 장식 |
| 배경 | 세계관 구성 요소 | 추적 불필요 | 가구, 건물 구조물 |

### 1-2. 유형별 분류

```
무기/방어구 -- 전투 씬에서 존재 여부 필수 확인
도구/열쇠 -- 퍼즐/진행에 필요한 물건
문서/서신 -- 정보 전달 매체, 지식 매트릭스와 연동
의복/장신구 -- 캐릭터 식별, 위장/변장 시 중요
화폐/보석 -- 경제 활동 추적
마법 물품 -- 능력/제한/비용 정의 필수
음식/소모품 -- 소비 시 소멸, 보유량 추적
```

---

## 2. 아이템 라이프사이클

### 2-1. 상태 변화

```
생성 -> 획득 -> 사용 -> [변형/파손/분실/양도/파괴]
```

모든 상태 변화는 `item-tracker.md`와 `novel-brain.json`에 기록한다.

### 2-2. 상태 코드

| 상태 | 설명 |
|------|------|
| 정상 | 원래 기능 유지 |
| 파손 | 부분적 손상, 수리 가능 |
| 분실 | 소유자가 위치 모름 |
| 파괴 | 회복 불가 |
| 변형 | 형태/기능 변경 |
| 봉인 | 사용 불가 상태 |

---

## 3. 소지품 관리 (인벤토리)

### 3-1. 캐릭터별 인벤토리

각 캐릭터가 현재 소지한 아이템 목록을 관리한다.

```
[캐릭터명]
현재 소지품:
- [아이템1] (상태: 정상)
- [아이템2] (상태: 파손)

이전에 소지했던 아이템:
- [아이템3] -> 5장에서 B에게 양도
```

### 3-2. 소지품 변동 규칙

- 아이템 이동(A -> B) 시 양쪽 인벤토리 업데이트 필수
- 아이템 사용 시 현재 소지 여부 먼저 확인
- 전투/이동 씬에서 무기/장비 소지 여부 확인
- 의복 변경 시 이전 의복의 행방 기록

---

## 4. 씬별 아이템 체크

### 4-1. 씬 시작 시 확인

씬 작성 전 다음을 확인한다:
- 등장 캐릭터들의 현재 소지품
- 장소에 존재하는 아이템
- 이전 씬에서 이동/변경된 아이템

### 4-2. 씬 종료 시 업데이트

씬 작성 후 다음을 업데이트한다:
- 이동된 아이템의 새 소유자/위치
- 상태가 변경된 아이템
- 새로 등장한 아이템
- 소멸된 아이템

---

## 5. 복선과 아이템

### 5-1. 체호프의 총

1막에서 벽에 걸린 총은 3막에서 발사되어야 한다.
- 의미 있게 등장한 아이템은 플롯에서 활용되어야 한다
- 활용 계획이 없는 아이템을 의미 있게 묘사하면 독자 기대를 배신한다
- `foreshadow-tracker.md`에 아이템 관련 복선을 연결한다

### 5-2. 맥거핀

플롯을 추동하지만 그 자체로는 중요하지 않은 아이템:
- 캐릭터들이 추구하는 목표물
- 내용보다 존재 자체가 중요
- 독자가 아이템의 본질을 알 필요 없을 수 있음

---

## 6. 일관성 검증

### 6-1. 자동 검증

novel-brain.json의 `item-location` 검증 규칙:
- 아이템이 2곳 이상에 동시 존재 감지
- 소유자가 없는 핵심 아이템 경고
- 파괴/분실된 아이템의 재등장 감지

### 6-2. 주간 점검

- 핵심 아이템의 현재 위치/상태 전수 확인
- 5챕터 이상 언급되지 않은 핵심 아이템 경고
- 캐릭터 인벤토리 정합성 확인
