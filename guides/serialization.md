# 웹소설 연재 가이드

웹소설 플랫폼에서의 연재를 체계적으로 관리한다.
버퍼 관리, 회차 구조, 독자 유지, 수익화를 포함한다.

---

## 1. 연재 기본 설정

### 1-1. 플랫폼 선택

| 플랫폼 | 특성 | 적합 장르 |
|--------|------|----------|
| 카카오페이지 | 유료 모델, 대중성 | 로맨스, 판타지, 무협 |
| 네이버시리즈 | 네이버 유입, 유료 | 전 장르 |
| 문피아 | 남성향 커뮤니티 | 판타지, 게임, SF |
| 조아라 | 자유 연재, 진입 용이 | 연습, 실험 |
| 리디 | 프리미엄 시장 | 로맨스, BL/GL |
| 노벨피아 | 자유 연재, 성장세 | 판타지, SF, 현판 |

### 1-2. 연재 스케줄

| 빈도 | 장점 | 단점 | 권장 버퍼 |
|------|------|------|----------|
| 매일 | 최대 노출, 습관 형성 | 높은 생산 부담 | 10회분+ |
| 주3회 | 균형적 | 중간 부담 | 5회분+ |
| 주2회 | 품질 확보 | 독자 이탈 가능 | 3회분+ |
| 주1회 | 최소 부담 | 낮은 노출 | 3회분+ |

## 2. 회차 구조

### 2-1. 기본 분량

| 플랫폼 | 권장 글자 수 | 비고 |
|--------|------------|------|
| 대부분 | 3,000-5,000자 | 가장 범용적 |
| 카카오페이지 | 4,000-6,000자 | 유료 가치감 |
| 문피아 | 5,000-8,000자 | 장문 선호 |

### 2-2. 회차 내 구조

```
[훅 오프닝] 첫 3-5줄 -- 즉시 흥미 유발
[전개] 사건/갈등 진행
[미니 정점] 회차 중간 텐션 피크
[엔딩 훅] 마지막 3-5줄 -- 다음 회 견인
```

### 2-3. 엔딩 훅 유형

| 유형 | 예시 |
|------|------|
| 위기 절단 | "칼날이 목 앞에서 멈췄다. 그리고--" |
| 폭로 직전 | "그 사람의 정체는, 바로--" |
| 결정 순간 | "두 가지 선택지가 남았다." |
| 반전 도입 | "죽었다고 생각한 사람이 문 앞에 서 있었다." |
| 미스터리 심화 | "하지만 진짜 문제는 그게 아니었다." |

## 3. 버퍼 관리

### 3-1. 버퍼 원칙

- 연재 시작 전 최소 목표 버퍼 확보
- 버퍼 3회분 미만 시 위기 경보
- 버퍼 소진 전 계획적 휴재 공지

### 3-2. 생산 속도 관리

novel-brain.json의 `serializationTracking.productionSpeed`에서 추적:
- 일일 작성량
- 주간 완성 회차 수
- 평균 회차 길이

생산 속도 < 연재 속도이면 버퍼가 줄어든다. 조기 경보 필수.

---

## 4. 독자 유지 전략

### 4-1. 이탈 방지

| 이탈 원인 | 대응 |
|----------|------|
| 전개 늘어짐 | 5회차 내 새로운 갈등 도입 |
| 캐릭터 매력 부족 | 빛나는 순간(shining moment) 정기 배치 |
| 예측 가능한 전개 | 10-15회차마다 소반전 |
| 불쾌한 전개 | 장르 약속 유지, 과도한 고통 절제 |

### 4-2. 독자 참여

- 댓글 반응 분석 -> 인기 캐릭터/전개 파악
- 독자 예측 추적 -> 너무 쉽지 않게 조절
- 팬 서비스 (인기 캐릭터 추가 등장) vs 작품 무결성 균형

---

## 5. 수익화 모델

### 5-1. 모델 유형

| 모델 | 설명 |
|------|------|
| 무료 | 광고 수익, 독자 확보 목적 |
| 부분유료 | 초반 무료 + 이후 유료 |
| 유료 | 전회차 유료 (이미 팬덤이 있을 때) |
| 후원 | 자발적 후원 (텀블벅, 패트리온) |

### 5-2. 무료-유료 전환 시점

- 최소 20-30회차 무료 공개 후 유료 전환
- 전환 직전 회차에 강력한 훅 배치
- 전환 시점이 아크 중간이면 효과적

---

## 6. novel-brain.json 연동

연재 데이터는 `serializationTracking` 섹션에서 관리한다.
- `buffer`: 현재 버퍼 회차 수
- `productionSpeed`: 생산 속도 지표
- `monetizationModel`: 수익화 모델 설정
- `readerMetrics`: 독자 지표 (구독, 조회, 유지율)
- `episodeHistory`: 발행 이력

`/novel serialize` 명령으로 연재 상태를 관리할 수 있다.
