# 플롯 구조 가이드

소설의 서사 구조를 설계하고, 복선과 서브플롯을 체계적으로 관리한다.

---

## 1. 구조 모델

### 1-1. 3막 구조

| 막 | 비율 | 핵심 기능 | 포함 요소 |
|----|------|----------|----------|
| 1막 (설정) | 25% | 세계/인물/갈등 제시 | 일상, 촉매 사건, 1차 전환점 |
| 2막 (대립) | 50% | 갈등 심화, 시련 | 상승, 중간점 반전, 하강, 최저점 |
| 3막 (해결) | 25% | 클라이맥스, 해소 | 2차 전환점, 절정, 결말 |

### 1-2. 기승전결

| 단계 | 비율 | 기능 |
|------|------|------|
| 기 (도입) | 25% | 상황 설정, 인물 소개 |
| 승 (전개) | 25% | 사건 발전, 갈등 확대 |
| 전 (전환) | 25% | 예상치 못한 전환, 반전 |
| 결 (결말) | 25% | 모든 스레드 수렴, 마무리 |

### 1-3. 영웅의 여정 (12단계)

```
일상 세계 -> 모험의 소명 -> 소명의 거부 -> 멘토와의 만남
-> 첫 관문 통과 -> 시련/동맹/적 -> 가장 깊은 동굴 접근
-> 시련 -> 보상 -> 귀환의 길 -> 부활 -> 영약과 귀환
```

### 1-4. 웹소설 구조

| 단위 | 분량 | 기능 |
|------|------|------|
| 회차 | 3,000-5,000자 | 독립적 미니 아크 + 엔딩 훅 |
| 에피소드 | 10-20회차 | 하나의 갈등 사이클 |
| 시즌 | 50-100회차 | 주요 아크 완결 |
| 완결 | 200-500회차 | 전체 서사 종결 |

각 회차는 반드시 다음 회를 읽게 만드는 훅으로 끝난다.

## 2. 플롯 설계 도구

### 2-1. 로그라인 공식

```
[주인공]이 [촉매 사건]으로 인해 [목표]를 추구하지만,
[장애물/적대자]가 가로막아 [핵심 갈등]에 빠진다.
[테마적 딜레마]를 해결하지 못하면 [위험/대가]를 치른다.
```

### 2-2. 갈등 계층

| 수준 | 설명 | 예시 |
|------|------|------|
| 내적 갈등 | 캐릭터 내면 | 두려움 vs 의무 |
| 대인 갈등 | 캐릭터 간 | 주인공 vs 적대자 |
| 사회 갈등 | 개인 vs 체제 | 반란, 사회 변혁 |
| 환경 갈등 | 개인 vs 자연/운명 | 생존, 재해 |

좋은 플롯은 최소 2개 수준의 갈등을 동시에 운용한다.

### 2-3. 이해관계 상승 (Escalation)

```
개인적 위험 -> 가까운 사람 위험 -> 공동체 위험 -> 세계적 위험
```

이해관계는 단조 증가해야 한다. 중간에 낮아지면 긴장이 풀린다.

## 3. 복선 관리

### 3-1. 복선 유형

| 유형 | 설명 | 회수 시점 |
|------|------|----------|
| 직접 복선 | 명시적 단서 제시 | 가까운 시점 (5챕터 이내) |
| 간접 복선 | 은유/상징으로 암시 | 먼 시점 (10챕터+) |
| 구조적 복선 | 패턴/반복 구조 | 클라이맥스 |
| 상징적 복선 | 모티프를 통한 암시 | 테마 해소 시점 |

### 3-2. 복선 관리 규칙

1. 심을 때: foreshadow-tracker.md에 등록 (장, 씬, 원문, 유형, 예정 회수 시점)
2. 단서 추가 시: 기존 복선에 추가 단서 연결
3. 회수 시: 회수 장/씬/원문 기록, 상태를 "회수완료"로 변경
4. 주간 점검: 미회수 복선 중 예정 시점 초과 항목 경고

### 3-3. 복선 밀도 가이드

| 장르 | 권장 밀도 (장당) |
|------|-----------------|
| 미스터리/스릴러 | 3-5개 |
| 판타지/SF | 2-3개 |
| 로맨스 | 1-2개 |
| 문학 | 1-3개 (상징적 위주) |
| 웹소설 | 1-2개 (회차당) |

## 4. 서브플롯 관리

### 4-1. 서브플롯 유형

| 유형 | 기능 | 메인플롯 연결 |
|------|------|--------------|
| 캐릭터 서브플롯 | 인물 깊이 확보 | 캐릭터 아크를 통해 |
| 로맨스 서브플롯 | 감정적 투자 유도 | 주인공 동기에 영향 |
| 미스터리 서브플롯 | 긴장감 유지 | 단서가 메인에 합류 |
| 테마 서브플롯 | 주제 다각적 탐구 | 주제적 공명 |

### 4-2. 서브플롯 규칙

- 모든 서브플롯은 메인플롯과 연결점이 있어야 한다
- 클라이맥스 전에 대부분의 서브플롯을 해소한다
- 서브플롯은 메인플롯의 텐션을 희석하지 않아야 한다
- 동시에 활성인 서브플롯: 단편 1개, 중편 2개, 장편 3-4개

## 5. 챕터 아웃라인

### 5-1. 챕터 설계 요소

각 챕터는 다음을 포함해야 한다:

```
목적: 이 챕터가 전체 플롯에서 수행하는 기능
갈등: 이 챕터의 핵심 긴장
변화: 챕터 시작과 끝에서 달라진 것
훅: 다음 챕터로 이어지는 견인력
```

### 5-2. 씬 목록

각 챕터는 2-5개 씬으로 구성하며, 각 씬은:
- POV (시점 캐릭터)
- 시간/장소
- 등장인물
- 씬 목표
- 갈등/긴장
- 결과

를 사전에 정의한다. `references/templates/project/03-plot/scene-checklist.md` 참조.

## 6. 전환점 (Turning Point) 설계

### 6-1. 필수 전환점

| 위치 | 전환점 | 기능 |
|------|--------|------|
| 1막 끝 | 1차 전환점 | 주인공이 돌아갈 수 없는 선택 |
| 2막 중간 | 중간점 반전 | 상황이 근본적으로 바뀜 |
| 2막 끝 | 최저점 (All Is Lost) | 가장 절망적인 순간 |
| 3막 시작 | 2차 전환점 | 최종 대결 시작 |
| 3막 중반 | 클라이맥스 | 핵심 갈등의 결정적 해소 |

### 6-2. 반전 설계 원칙

- 놀랍지만 불가피해야 한다 (사후적으로 납득 가능)
- 복선이 뒷받침해야 한다
- 캐릭터의 핵심 특성에서 비롯되어야 한다
- 이해관계를 높여야 한다 (낮추면 안 된다)

## 7. novel-brain.json 연동

플롯 관련 데이터는 `plotTracking` 섹션에서 관리한다.
- `structure`: 현재 구조 모델과 진행 위치
- `mainPlot`: 중심 갈등과 해소 계획
- `subplots`: 서브플롯 목록과 상태
- `foreshadowing`: 복선 전체 목록
- `chapterOutlines`: 챕터별 설계

`/novel outline` 명령으로 아웃라인을 생성/수정하면 자동으로 plotTracking에 반영된다.
