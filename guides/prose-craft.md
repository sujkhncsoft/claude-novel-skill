# 산문 기법 가이드

문장 단위의 품질을 높이고, 문체 일관성을 유지하며, 클리셰를 감지한다.

---

## 1. 문체 분석 지표

### 1-1. 정량 지표

| 지표 | 설명 | 이상 범위 |
|------|------|----------|
| 수사 밀도 | 비유/은유 수 / 1000단어 | 5-15 (장르별 조정) |
| 문장 길이 다양성 | 짧은/중간/긴 문장 비율 | 고르게 분포 |
| 대화:서술 비율 | 대사 분량 / 전체 분량 | 30-60% (장르별) |
| Show vs Tell 비율 | 장면 묘사 vs 설명 | 70:30 이상 |
| 감각 묘사 밀도 | 오감 묘사 / 페이지 | 2-5회 |
| 부사 빈도 | 부사 수 / 1000단어 | 10 이하 |
| 수동태 비율 | 수동 문장 / 전체 문장 | 15% 이하 |
| 고유 단어 비율 | 고유 단어 수 / 전체 단어 | 60%+ |

### 1-2. 정성 지표

- 음성 일관성: 시점 캐릭터에 따른 문체 변화
- 톤 일관성: 장면 분위기와 문체의 조화
- 리듬: 문장 길이의 음악적 흐름

## 2. Show, Don't Tell

### 2-1. 원칙

```
[Tell] 그는 화가 났다.
[Show] 그의 주먹이 허벅지 위에서 떨렸다. 턱 근육이 단단하게 경직되었다.
```

### 2-2. 감정 표현 가이드

| 감정 | Tell (금지) | Show (권장) |
|------|-----------|-----------|
| 분노 | "화났다" | 신체 반응 (떨림, 경직, 열감) |
| 슬픔 | "슬펐다" | 행동 (시선 회피, 목소리 변화) |
| 두려움 | "무서웠다" | 감각 (심장, 호흡, 동공) |
| 기쁨 | "기뻤다" | 동작 (미소, 걸음걸이, 목소리) |

### 2-3. 예외

Tell이 더 효과적인 경우:
- 빠른 전개가 필요한 전환 구간
- 시간 경과를 압축할 때
- 의식적 스타일 선택 (하드보일드 등)

## 3. 문장 리듬

### 3-1. 길이 변주

```
짧은 문장은 충격을 준다.
그것은 마치 망치로 한 번 내리치는 것과 같아서, 독자의 시선을
멈추게 하고, 이전에 흘러가던 긴 문장들의 부드러운 리듬을 깨뜨리며,
갑작스러운 정적을 만들어낸다. 그리고 다시 짧게. 끊는다.
```

### 3-2. 리듬 규칙

- 긴장 장면: 짧은 문장 위주 (급박함)
- 묘사 장면: 긴 문장 위주 (몰입감)
- 대화 장면: 대사 길이로 캐릭터 성격 반영
- 단조로움 방지: 3문장 이상 같은 길이 연속 금지

## 4. 클리셰 감지

### 4-1. 흔한 묘사 클리셰

| 카테고리 | 클리셰 표현 | 대안 |
|----------|-----------|------|
| 눈 | "보석 같은 눈" | 구체적 특징으로 대체 |
| 미소 | "해처럼 밝은 미소" | 미소의 고유한 특성 |
| 심장 | "심장이 뛰었다" | 구체적 신체 감각 |
| 시간 | "시간이 멈춘 듯" | 감각적 경험으로 전환 |
| 감정 | "가슴이 찢어지는" | 구체적 행동/감각 |

### 4-2. 구조적 클리셰

| 카테고리 | 클리셰 | 대안 |
|----------|--------|------|
| 오프닝 | 꿈에서 깨어남 | In medias res (사건 중간) |
| 회상 | 긴 플래시백 | 현재 행동에 과거 녹이기 |
| 설명 | 거울 보며 자기 묘사 | 타인 시선/행동 통해 드러냄 |
| 엔딩 | "그것은 시작에 불과했다" | 구체적 다음 행동/상황 |

## 5. 시점 관리

### 5-1. 시점 유형

| 시점 | 특성 | 적합 상황 |
|------|------|----------|
| 1인칭 | 친밀감, 주관적 | 내면 탐구, 미스터리 |
| 3인칭 제한 | 균형, 한 캐릭터 시점 | 가장 범용적 |
| 3인칭 전지 | 넓은 시야, 다각적 | 서사시, 대하소설 |
| 2인칭 | 실험적, 몰입 | 특수 효과 |

### 5-2. 시점 일관성 규칙

- 선택한 시점을 씬 단위로 유지한다
- 시점 전환은 챕터 또는 명시적 구분선에서만
- 시점 캐릭터가 모르는 것은 서술하지 않는다
- 시점 캐릭터의 어휘/사고방식이 서술에 반영된다

---

## 6. 대화 기법

### 6-1. 좋은 대화의 조건

- 정보 전달 + 캐릭터 드러내기 + 갈등 진행 (최소 2가지 동시)
- 실제 대화처럼 불완전하고, 간접적이다
- 서브텍스트가 있다 (말하는 것 != 의미하는 것)

### 6-2. 대화 태그

```
[좋음] "알겠어." 그가 말했다.
[나쁨] "알겠어." 그가 차갑게 내뱉었다.
[더 좋음] "알겠어." 그가 컵을 내려놓았다.
```

- "말했다/물었다"가 가장 투명한 태그
- 행동 묘사가 감정을 전달한다
- 부사가 붙은 태그("화나게 말했다")는 대부분 불필요

## 7. novel-brain.json 연동

문체 분석 결과는 `proseAnalysis` 섹션에 저장된다.
`/novel prose [챕터]` 명령으로 특정 챕터의 문체를 분석할 수 있다.
